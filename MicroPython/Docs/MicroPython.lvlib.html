MicroPython.lvlib:TxRx.vi

This VI will transmit a message over the serial port and optionally wait for a response from the target device.

If waiting for a reponse, we will listen as long as the device is sending data or until with have received more than some maximum threshold (set in the block diagram. See MaxRxLength).

After each bundle of data is received, we will wait for a period equal to "Wait Before Read (Ms)" before trying to read more.

If we have waited for "Wait Before Read (Ms)" and we notice timeout is reached or we have at least one byte received, we will assume the device is done sending messages, and we will leave.

MicroPython.lvlib:SendCommand.vi

Sends a command to a micropython target. The VI adds "\r" to a non control message.

Timeout (Sec) indicates that if, in a new receive cycle, we haven't received from the device and this timeout has been reached since we started listening, we will assume that the device is done talking.

Wait Before Read (Ms) indicates how long we should wait before checking the receive buffer at the beginning of a receive cycle.
